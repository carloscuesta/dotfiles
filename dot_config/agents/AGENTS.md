# Carlos agent instructions

These are common instructions for Carlos agents across all scenarios.

## General guidelines

- Never write any comments unless the logic is non-obvious or additional context is needed.
- Do not add `Co-Authored-By` to git commits.
- Never manually modify files that are automatically generated.
- All code changes must be covered with a set of tests, covering all possibles branches and outputs.
- When making technical decisions, do not give much weight to development cost. Instead, prefer quality, simplicity, robustness, scalability and long term maintainability.

## Voice profile

When you are talking or writing on behalf of Carlos using his identity, read [@VOICE.md](~/.config/agents/VOICE.md) to see how Carlos talks.

## Opinions

When you are working on something that would benefit from being informed by Carlos viewpoints, read [@OPINIONS.md](~/.config/agents/OPINIONS.md) to understand what Carlos believes.
